1. 基础背景与文本色（冷峻基调）
主背景色 (Background)： #141517 或 #1A1D20（不要用纯黑 #000000，深灰黑更有质感，且带有一丝冷蓝/冷绿倾向）。
次级背景/面板色 (Surface)： #222529 或 #2A2D32（用于卡片、弹窗的底色）。
主文本色 (Text Primary)： #E8E8E8 或 rgba(255, 255, 255, 0.9)（灰白色，降低刺眼感）。
次级文本色 (Text Secondary)： #8F9399 或 rgba(255, 255, 255, 0.5)（用于说明文字、时间戳、坐标等）。
2. 核心强调色（灵魂点缀）
源石黄/警告黄 (Accent Yellow)： #FFD800 或 #F4C542（最经典的方舟黄，用于主按钮、高亮边框、重要提示）。
危险红/医疗红 (Accent Red)： #E60012 或 #FF3B30（用于错误提示、删除操作、敌人/危险标识）。
理智蓝/科技蓝 (Accent Blue)： #00A0E9 或 #4FC3F7（用于信息提示、链接、科技感进度条）。
3. 边框与分割线
常规边框： rgba(255, 255, 255, 0.1) 或 #33373D。
高亮边框： 使用强调色（如黄色）并配合 box-shadow 产生微弱的发光效果。
二、 设计语言 (Design Language)
明日方舟的UI风格可以总结为 “战术工业风” (Tactical Industrial) 与 “HUD（平视显示器）拟物”。
1. 几何切割与斜角 (Geometric Clipping)
这是方舟UI最核心的视觉特征。放弃传统的圆角（Border-radius），大量使用直角和斜切角。
实现方式： 强烈建议使用 CSS 的 clip-path: polygon(...) 来实现缺角矩形，而不是用图片。这样才符合“即插即用”和轻量化的要求。
常见形态： 左上/右下缺角、右侧斜切、顶部带有小三角形缺口。
2. 高信息密度与模块化 (High Density & Modular)
面板嵌套： 大卡片里嵌套小卡片，通过不同层级的背景色和细边框区分。
网格与对齐： 严格遵循网格系统，元素对齐必须极其精准，营造“战术面板”的严谨感。
3. 装饰性排版 (Decorative Typography)
中英混排与等宽字体： 标题使用粗体无衬线黑体（如 思源黑体 Heavy 或 OPPOSANS），正文使用常规字重。
背景装饰： 在标题或卡片背景中，加入巨大的、低透明度的英文字母、数字编号（如 SYS-01, OP. 42）、坐标轴或条形码。
状态指示： 大量使用小圆点、细线条、十字准星、进度条来装饰空白区域。
4. 微动效与故障艺术 (Micro-interactions & Glitch)
悬停反馈： 鼠标悬停时，边框颜色从灰色变为黄色，伴随轻微的 transform: translate 位移，或者内部背景色微调。
故障闪烁： 在页面加载或重要提示出现时，可以使用 CSS 动画实现轻微的透明度闪烁或位移（Glitch effect），但切忌过度，以免影响阅读。
